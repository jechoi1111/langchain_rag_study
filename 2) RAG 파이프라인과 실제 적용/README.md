# RAG 파이프라인과 실제 적용

## RAG 파이프라인

- RAG는 문서를 숫자로 변환하여 저장하는 인덱싱, 검색하고 답변을 생성하는 검색/생성 두가지 구조.

### 데이터 인덱싱

문서 -> 분할된 문서(Chunk) -> 벡터 임베딩 -> 벡터 DB

### 데이터 검색 및 생성

질문 -> 벡터 임베딩 -> 벡터 DB 검색(Retrieval) -> 유사 문장 출력(Top-K chunks) -> LLM -> 답변 생성

## RAG의 핵심 컴포넌트

- RAG를 구성하기 위해서는 문서 로더, 임베딩 모델, 벡터 DB, LLM 등 다양한 컴포넌트가 필요.

  - 벡터 DB : Pinecone, ChromaDB, Milvus, Qdrant
